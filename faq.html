Frequently asked questions

Question 1. How do I use this extension?

Answer Let's say, for example, that you have opened a web page with many PDF resources. The most basic workflow is to open the extension popup using the browser toolbar icon and click the Load page links button. This usually fills the resource list with all sorts of links, not only the desired PDFs. The next step is to filter the list so that only the desired PDFs remain. Enter pdf in the extensions field. Only PDF resources will now be displayed. If the list is still too large and you want only a smaller subset, continue filtering by entering relevant terms in the text filter field. Finally, check the desired items individually or all at once and start downloading immediately using the corresponding button in the bottom-right corner.

The downloading items are now visible in the Downloads tab of the popup. You can manage this list in the usual manner: pause, resume, open, or remove individual items.

Question 2. Are there alternative workflows?

Answer The general idea of the extension is to collect links in the resource list, optionally filter them, and finally add checked items to the download queue. The collection stage has several variations:

  • Use the web page context menu: right-click individual links or a larger selected text region.
  • Use the special tabs dialog: collect links from the active tab | all tabs | right tabs | left tabs, optionally download the collected links immediately, or close the associated tabs when the operation is complete.
  • Add links from clipboard text using the dedicated menu. Each URL must be on a separate line.

Question 3. The resource list has too many links and I find it difficult to recognize the desired ones!

Answer By default, the resource list displays the estimated filenames of the collected links. These names are usually different from those displayed on the web page. There are many possible strategies here:

  • If the desired items are images, it may be better to use the thumbnails view. There is a special toggle button for this. Here the images can be sorted in descending order by size or filtered by width and height.
  • If you know the file extensions of the desired items, use them in the extensions filter box.
  • Sometimes it may be better to select a text zone on the web page (including relevant links) and use the context menu to extract only links from this selected text range.
  • More generally, every item has textual information associated with it. This information comes from the web attributes of that item, more precisely:
    • the URL (the link address) visible in the status bar when you hover over the link on the web page
    • the filename, usually included in the URL
    • the link text (as you see on the web page)
    • the title text from the tooltip displayed when you hover over the link on the web page
    • the alt text (alternative text, usually associated with images)

    You can use this information to identify specific terms associated with your desired items. Enter these terms in the text filter box using a simple syntax, for example url:word_1 text:word_2 title:word_3. Alternatively, use all:word if you are not sure which attribute contains the word.

  • Right-click any item in the resource list to display an informative popup with all the available information for that item.

Question 4. How do I set a custom download directory?

Answer The resource list tab contains a directory input field. Enter a folder in the form aaa/bbb/.../zzz, for example Docs or Docs/Math/Algebra. Every such directory is relative to the default download directory. This is a browser extension security limitation, so you can define only subdirectories of the main download directory. You can use the general browser settings to change the location of that main directory.

You can also prepare a custom list of favorite folders in the Download directories section of the extension options. Once defined, this list is available via autocomplete or by double-clicking the directory field.

Question 5. Why are there two add to queue buttons?

Answer After checking items in the resource list, you can send them either to the active queue or to the passive queue using the corresponding buttons. Sending items to the active queue means that downloading starts immediately or as soon as possible. This is the best option if you are content with the default names. Sometimes, however, it is better to rename queued items before downloading them. You can do this individually using the edit button associated with each item in the download list, or in batches using the multi-rename dialog.

Question 6. I want to download files from the resource list, but I prefer the names displayed on the original web page. How can I use them?

Answer By default, items in the resource list are displayed with their estimated filenames, using the {name}.{ext} name mask. Click the name header of the resources table to choose another attribute for display: URL, title, text, or alt text. The most obvious candidate is the text attribute, which usually corresponds to what you see on the web page. In this case, items are displayed with the {text}.{ext} name mask. Sending them to the queue preserves this displayed mask.

Once items are added to the passive queue, you can rename them in more advanced ways. Select items and use the multi-rename dialog to rename them in batches. Here you can combine several types of masks and perform other batch operations, such as search and replace or setting directories.

Question 7. Why is the extension unable to download embedded videos?

Answer Simple Mass Downloader is designed for normal, direct file links. Embedded videos and streaming media use different delivery mechanisms and are outside its scope. For these resources, use a specialized video or media downloader.

Question 8. Why is the extension unable to download files from sites X, Y, or Z?

Answer The extension detects links exposed by the current page. It scans the page DOM, including accessible open shadow roots, but some sites expose resources in nonstandard ways. In other cases, the visible items are only placeholder links: the real URLs are generated on the fly, only after a user action. Such URLs cannot always be collected in advance.

One possible solution is a helper userscript designed specifically for that site. With a userscript manager such as Tampermonkey, a knowledgeable user can expose the relevant information using the special attributes data-smd-url and data-smd-name. Simple Mass Downloader recognizes these attributes and can use them for batch downloading.

Another option is to use an extension designed specifically for that site. Popular services often have specialized download extensions.

Question 9. Why does the extension sometimes detect only thumbnails instead of full-sized images?

Answer As in the previous question, the page may expose only thumbnail URLs. The full-sized image URLs may be generated dynamically or obscured by site-specific JavaScript. See the previous answer for possible solutions.

Question 10. I found a bug. How can I help?

Answer Simple Mass Downloader has many features and recently underwent a major internal migration to Chrome Manifest V3. Bugs and regressions are possible, especially on unusual sites or less common workflows.

Detailed bug reports are very helpful. A concrete example of a troublesome page or resource URL, together with clear reproduction steps, is much more useful than a general complaint such as "the extension did not work". This makes it possible to analyze the problem and, when feasible, fix it.
