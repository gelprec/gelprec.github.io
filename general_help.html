Thanks for using the Simple Mass Downloader extension!

Before using, please read the following advice/warnings:

  • First of all, please verify in your Chrome Download Settings that the following option: Ask where to save each file before downloading is NOT checked!
    Otherwise, a lot of "Save As" dialog boxes will pop up, defeating the main purpose of this extension (or any other mass downloading Chrome extension)!
  • The extension has recently been migrated to Chrome Manifest V3, so bug reports and suggestions are welcome (use this web page )
  • Development is mainly done on Windows, with limited testing on Linux (latest Ubuntu). The extension may also work on macOS, but it has not been tested there.
  • This extension is a general file downloader. It's not intended for downloading streaming media or embedded video content. For these resources, use a specialized browser extension, such as a video downloader.
  • The extension doesn't work in private browsing mode.
  • Inside the extension, you may define custom download folders, but these are relative to the browser's default download directory; this is a general browser extension security limitation.

 General Usage

Unlike external software that usually uses only the information contained in the link URL, this mass downloader can take advantage of the various text attributes of the links on a web page ( URL, title, text, alt text, etc.) and use them in various contexts: when filtering the resource list, renaming resources in batches, or defining rules for automatic folders. On the other hand, being just a web extension, it only uses the browser's built-in download engine and consequently it lacks some useful features: multithreading capabilities, bandwidth control, the ability to choose an arbitrary download directory or rename files after downloading. Also, an external downloader usually has better resume support and better detection of previously downloaded files. Many of these limitations are a consequence of the modern browsers' strong security policies. In general, preparing the list of download links (i.e. link collection, filtering and initial renaming) is more natural in the browser context, while the actual process of downloading may benefit more from an external downloader (but, fortunately, in 99% of cases the built-in browser download engine is more than enough).

The main workflow involves three steps:

  • Add links, images, and other resources from the current page to the resource list and mark the desired items after optional filtering.
  • Add the marked files to the download queue. These queued files may optionally be renamed (individually or using the available multi-rename dialog)
  • Finally, select queued files, either all of them or only a range, and start downloading them.

Unlike some other bulk download managers, this extension does not automatically load links from the active tab. You explicitly add links from the active tab | all tabs | left or right tabs, or using the web page's context menu. In this way, the list is successively augmented and after this initial collecting stage, you may filter, rename, or mark selected items for download.

A lighter workflow is available if you select links via the page context menu or right-click a blank point on the page: in this case, the extension opens a quick download dialog with minimal filtering options and a Download now button. This workflow lacks advanced filtering/renaming options but may be appropriate in many situations.

Resources tab

The main Load page links button empties the current list and fills it using only links from the active tab. There are many other ways to add resources, and all these additional methods add links to the current resource list:

  • via the page context menu: add individual links/images or multiple links from custom text selections or from the whole page.
  • via a dedicated tabs button that opens a special dialog where you select tabs (active tab, all tabs, left or right tabs) and optionally filters page items by various criteria. Finally, the found links are added to the resource list and are ready for further operations.

Clicking the resource list name header will open a popup where you may quickly select the current attribute that generates the displayed names for the resource list ( filename/URL/title/text/alt text). This may be useful for identifying the desired files.

Note that you may get all attributes/information available for an individual item by right-clicking an item. This will open an informative popup with all attributes available for that item ( URL/title/text, etc.), including a thumbnail view for image items. Here you may also open the link in a new tab or copy its URL.

The resource list may be filtered by extension or by item text (URL, link text, title, alt text). Note that the text filter field uses whitespace as an AND operator, in contrast to the extensions filter field, where whitespace acts as an OR operator. You can use the special syntax source:term, where source is all|url|name|text|title|alt, or the shorter corresponding variants a|u|n|x|t|l. Negative forms are also possible using the - operator.

For advanced users, there is also a special regular-expression mode: to enter that mode the text must begin with r: and the rest of the string is interpreted as a regular expression. The same special filtering rules (for the text filter field) apply to Tabs scan / Extra scan dialogs.

Filtering examples:

  • for the extensions field: jpg png filters resources that have either a jpg or png extension.
  • for the text field:
    • term_1 term_2 filters resources for which the currently displayed attribute contains term_1 AND term_2.
    • all:term_1 filters resources for which at least one attribute contains term_1. Shorter version a:term_1.
    • url:term_1 text:term_2 filters resources that have term_1 somewhere in their URL AND term_2 somewhere in their link text. Shorter version: u:term_1 x:term_2.
    • url:term_1 -text:term_2 filters resources that have term_1 somewhere in their URL and NOT term_2 in their link text. Shorter version: u:term_1 -x:term_2 (note that the search terms do not have spaces).

For images, there is also a special thumbnails view, where items may be sorted in descending order by size.

In the Resources tab, you can also set a common download directory, written in the form aaa/bbb/.../zzz (this can only be specified as a subfolder of the default user download directory; it's a browser extension security limitation)

After checking the desired files, click the Add to queue button to continue to the Downloads tab.

Downloads tab

Here you may start downloading individual items, or select multiple items and finally click the Start selected button. Selection may be done using ctrl + click for individual items or shift + click for range selections.

Before downloading, you may change the filename/directory for individual items (every item that has not started downloading has an edit button), or rename selected files. The default file name is {name}.{ext} (for Firefox) or {auto} (for Google Chrome) but other masks may be used too: {url}, {title}, {alt}, {text}, {counter}, {host}.

The multi-rename and batch actions dialog also has a search/replace section; here it's possible to use regular expressions. In this case, the replacement string may contain special patterns $1, $2... etc. where $1 represents the first parenthesized submatch string, and so on. Other patterns are also available; for more details see the Specifying a string as a parameter section from here.

Note that all renaming procedures are available only for items that have not started downloading.

You may set the maximum number of active servers and the number of active downloads per server.

The context menu for an item contains the usual actions for this item: cancel/remove/retry/open file/open directory/pause. The special combination ctrl + right click reveals information associated with that item ( URL/title/text, etc.).

Favorite and recent folders

There are some contexts where you may set a download directory (inside the resource list view or quick-dialog/multi-rename/edit-item popups). You may explicitly enter the folder name in the corresponding input text field or, sometimes more conveniently, double-click this input field and select from a popup list either a recently used folder or a favorite folder. The favorite folder list can be edited on the extension's general settings page.

Automatic Folders

The settings dialog has an Automatic Folders section; here you may define custom folders where downloads can be automatically directed based on user-defined rules.

Every automatic folder has (at least) an associated rule set. A resource will be saved in this folder only if it satisfies all the rules in the rule set. A rule must be written in the form

keyword: word1, word2, ...

where keyword may be all|ext|url|name|host|text|alt|title, for example ext: jpg, png, bmp. So a resource satisfies this sample rule only if its file extension is jpg or png or bmp.

In this dialog, you may define new folders or select previously defined folders for editing. A new folder is written in the form aaa/bbb/.../zzz. Please note that every folder is defined relative to the default download directory (this is a browser extension security limitation). So a folder defined as John/Music is, in fact, a folder named Music located inside a folder named John, and the latter is created inside the default download directory.

A folder can be removed. This only removes it from the automatic folders list; it does not delete it from disk.

Additional Automatic Folders tips:

  • A rule may also be specified as keyword_1, keyword_2, ...: word1, word2, .... This type of rule is satisfied if the value associated with keyword_1 ( OR the value associated with keyword_2) contains at least one word from the list word1, word2, ....
  • all: word1, word2, ... is a shortcut for ext, url, name, host, text, alt, title: word1, word2...
  • In the case of conflicting folders, the item's edit button is displayed in a warning color. The directory field in the edit popup has a dropdown arrow, allowing you to choose between these folders. If no directory is chosen, the default download directory will be used.

Additional features

Extra scan dialog

This feature is not exposed in the current preliminary MV3 interface. It may return in a future version.

When available, select some web-page resources in the list and the extension will try in the background to extract links from these pages according to the filtering criteria specified in Extra Scan dialog. Finally, the found links are inserted in the resource list below their "parent" item and are ready for further actions.

Sequential mode

There is also a special sequential mode for situations where the server does not allow multiple files to be downloaded in parallel. The files are downloaded one by one, with an interval between downloads. This value may be changed in the settings dialog.

External userscripts support

The extension detects links exposed by the current page. It scans the page DOM, including accessible open shadow roots, but some sites expose resources in nonstandard ways. In other cases, the visible items are only placeholder links: the real URLs are generated on the fly, only after a user action. Such URLs cannot always be collected in advance.

One possible solution is a helper userscript designed specifically for the site. With a userscript manager such as Tampermonkey, a knowledgeable user can build a userscript that facilitates subsequent downloading with Simple Mass Downloader. More precisely, the extension can recognize the special attributes data-smd-url and data-smd-name and use them for batch downloading. The helper userscript must expose the information required for download through these attributes.

Other Info

  • When defining directories (including automatic or favorite folders), the following masks can be used: {yyyy}, {yy}, {MM}, {dd}, {host}, {ww}, {url:n}, {ext}
  • By default, the resource list uses filenames as the displayed name for list items, but the you may change this in the settings dialog. These filenames aren't always very accurate (including file extensions) because the resource list is built using only information available on the current page. But for checked items, the extension tries to fetch supplementary metadata that may reveal more accurate naming data. Because the fetching process may be relatively expensive, it is not enabled by default for all items in the resource list.
  • After using Extra scan dialog, if the found links are selected for the multi-rename and batch actions dialog, extra masks are available: {pname}, {purl}, {ptitle}, {palt}, {ptext} where pname is the name of the parent item and so on.
  • In the Tabs scan / Extra scan dialogs, at least one filtering criterion must be specified to prevent the resource list from being overloaded.
  • Items in the download list view can be reordered using ctrl + up/down arrow keys.
  • The number of items per page may be set individually for the resource list, thumbnails grid, and download list. For example 200, 50, 50 instead of the default 200 (equivalent to 200, 200, 200)
  • You can directly import a list of URLs into the download list using the dedicated submenus from the hamburger menu in the Download List tab. The sources for this local list are the clipboard or, in Chrome, a local file. Note that the URLs must be separated by spaces or newlines.
  • You can also set a pattern URL using the dedicated submenu from the hamburger menu (Download List tab). For example http://example.com/image-[001:100].jpg
    These pattern URLs use the special batch descriptors [start:end] or [start:end:step] where
    • start/end are positive integers or ASCII letters
    • [optional] step is an integer, optionally negative
    Note that start/end must have the same type integer|upper|lower.
    More examples:
    Batch descriptor      Generated result
    [1:100] 1, 2, 3 ... 99, 100
    [001:100] 001, 002, ... 099, 100
    [1:100:3] 1, 4, 7 ... 100
    [100:1:-3] 100, 97, 94 ... 1
    [a:z] a, b, c ... z
    [A:G:2] A, C, E, G

    Comment: The syntax used for batch descriptors broadly follows the one defined by DownThemAll here.
